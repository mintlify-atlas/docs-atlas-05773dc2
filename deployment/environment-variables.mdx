## Overview

DRAIT Mini-MES uses environment variables for configuration. All variables are defined in a `.env` file in the project root.

## Setup

<Steps>
  <Step title="Copy Example File">
    ```bash
    cp .env.example .env
    ```
  </Step>

  <Step title="Edit Configuration">
    Open `.env` and update values for your environment:
    ```bash
    nano .env
    # or
    vim .env
    ```
  </Step>

  <Step title="Restart Services">
    After changing environment variables:
    ```bash
    docker compose down
    docker compose up -d
    ```
  </Step>
</Steps>

## Variable Reference

### Global Configuration

#### NODE_ENV

<ResponseField name="NODE_ENV" type="string" default="development" required>
  Runtime environment mode

  **Values:**
  - `development` - Development mode with verbose logging
  - `production` - Production mode with security validations

  <Warning>
    In production mode, the backend validates `JWT_SECRET` and refuses to start with weak default values.
  </Warning>
</ResponseField>

```bash .env
NODE_ENV=production
```

#### TZ

<ResponseField name="TZ" type="string" default="America/Buenos_Aires" required>
  Timezone for timestamps and date operations

  Use standard IANA timezone identifiers (e.g., `America/Buenos_Aires`, `America/New_York`, `Europe/London`).
</ResponseField>

```bash .env
TZ=America/Buenos_Aires
```

### Database Configuration

#### POSTGRES_DB

<ResponseField name="POSTGRES_DB" type="string" default="drait_mes" required>
  PostgreSQL database name
</ResponseField>

```bash .env
POSTGRES_DB=drait_mes
```

#### POSTGRES_USER

<ResponseField name="POSTGRES_USER" type="string" default="drait" required>
  PostgreSQL username

  <Info>
    This user is created automatically when the database container starts for the first time.
  </Info>
</ResponseField>

```bash .env
POSTGRES_USER=drait
```

#### POSTGRES_PASSWORD

<ResponseField name="POSTGRES_PASSWORD" type="string" default="drait123" required>
  PostgreSQL password

  <Warning>
    **CRITICAL:** Change this from the default value in production! Use a strong password with at least 16 characters.
  </Warning>
</ResponseField>

```bash .env
POSTGRES_PASSWORD=your_secure_password_here
```

#### POSTGRES_PORT

<ResponseField name="POSTGRES_PORT" type="number" default="5432">
  Port to expose PostgreSQL on the host machine

  <Note>
    This port is only used for external database access. Internal Docker services use port 5432 regardless of this setting.
  </Note>
</ResponseField>

```bash .env
POSTGRES_PORT=5432
```

#### DATABASE_URL

<ResponseField name="DATABASE_URL" type="string" required>
  Complete PostgreSQL connection string for Prisma

  **Format:**
  ```
  postgresql://<user>:<password>@<host>:<port>/<database>?schema=public
  ```

  <Warning>
    The host must be the Docker service name `drait-db` (or `db` in older configs), not `localhost`.
  </Warning>
</ResponseField>

```bash .env
DATABASE_URL=postgresql://drait:your_secure_password_here@drait-db:5432/drait_mes?schema=public
```

### Backend Configuration

#### BACKEND_PORT

<ResponseField name="BACKEND_PORT" type="number" default="3000" required>
  Port where the NestJS backend listens inside its container

  <Info>
    This is an internal port. External access goes through Nginx on `HTTP_PORT`.
  </Info>
</ResponseField>

```bash .env
BACKEND_PORT=3000
```

#### JWT_SECRET

<ResponseField name="JWT_SECRET" type="string" default="change_this_secret" required>
  Secret key for signing and verifying JWT tokens

  <Warning>
    **CRITICAL SECURITY SETTING:**
    - Must be at least 32 characters long
    - Use a cryptographically random string
    - Never commit this to version control
    - In production mode, the backend **will refuse to start** if this is set to a weak default value

    Weak values that are rejected:
    - `change_this_secret`
    - `secret`
    - `changeme`
    - `jwt_secret`
  </Warning>

  Generate a secure secret:
  ```bash
  # Using openssl
  openssl rand -base64 32

  # Using node
  node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
  ```
</ResponseField>

```bash .env
JWT_SECRET=your_long_random_secret_at_least_32_chars
```

The backend validates JWT_SECRET at startup:

```typescript apps/backend/src/main.ts
const WEAK_SECRETS = ['change_this_secret', 'secret', 'changeme', 'jwt_secret'];

if (process.env.NODE_ENV === 'production' && WEAK_SECRETS.includes(jwtSecret.toLowerCase())) {
  logger.error(
    '❌ [SECURITY] JWT_SECRET tiene un valor débil o por defecto. Definí un JWT_SECRET fuerte en el .env antes de iniciar en producción.'
  );
  process.exit(1);
}
```

#### JWT_EXPIRES_IN

<ResponseField name="JWT_EXPIRES_IN" type="string" default="8h" required>
  JWT token expiration time

  **Format:** Use time units like `1h`, `8h`, `24h`, `7d`

  **Recommendations:**
  - Development: `24h` or longer for convenience
  - Production: `8h` for security
  - High-security environments: `1h` or `2h`
</ResponseField>

```bash .env
JWT_EXPIRES_IN=8h
```

#### CORS_ORIGIN

<ResponseField name="CORS_ORIGIN" type="string" default="http://localhost" required>
  Allowed origin for Cross-Origin Resource Sharing

  **Examples:**
  - `http://localhost` - Local development
  - `http://192.168.1.100` - LAN access
  - `https://mes.example.com` - Production domain

  <Warning>
    Do not use `*` in production. Always specify the exact origin.
  </Warning>
</ResponseField>

```bash .env
CORS_ORIGIN=http://localhost
```

### Frontend Configuration

#### VITE_API_BASE_URL

<ResponseField name="VITE_API_BASE_URL" type="string" default="/api" required>
  Base URL for API requests from the frontend

  **Common values:**
  - `/api` - When using Nginx reverse proxy (recommended)
  - `http://localhost:3000/api` - Direct backend access in development

  <Info>
    With the Docker Compose setup, always use `/api` because Nginx handles routing.
  </Info>
</ResponseField>

```bash .env
VITE_API_BASE_URL=/api
```

### Infrastructure Configuration

#### HTTP_PORT

<ResponseField name="HTTP_PORT" type="number" default="80">
  Port to expose Nginx on the host machine

  <Note>
    If port 80 is already in use, change this to another port like `8080` or `3000`.
  </Note>
</ResponseField>

```bash .env
HTTP_PORT=80
```


## Environment Variable Loading

Docker Compose loads variables from `.env` automatically:

```yaml docker-compose.yml
environment:
  NODE_ENV: ${NODE_ENV}
  JWT_SECRET: ${JWT_SECRET}
  DATABASE_URL: ${DATABASE_URL}
```

<Warning>
  Variables prefixed with `VITE_` are embedded into the frontend build at compile time. Changing them requires rebuilding the frontend container.
</Warning>

## Troubleshooting

### Backend Refuses to Start

**Error:** `JWT_SECRET tiene un valor débil o por defecto`

**Solution:** Generate a strong JWT secret and update `.env`:
```bash
openssl rand -base64 32
```

### Database Connection Fails

**Error:** `Can't reach database server`

**Solution:** Verify `DATABASE_URL` uses the correct Docker service name:
- Correct: `postgresql://user:pass@drait-db:5432/drait_mes`
- Wrong: `postgresql://user:pass@localhost:5432/drait_mes`

### CORS Errors in Browser

**Error:** `Access to fetch at '...' from origin '...' has been blocked by CORS policy`

**Solution:** Set `CORS_ORIGIN` to match the URL in your browser's address bar:
```bash
CORS_ORIGIN=http://192.168.1.100
```

### Frontend Can't Connect to Backend

**Error:** Network errors when making API requests

**Solution:** Ensure `VITE_API_BASE_URL=/api` and rebuild the frontend:
```bash
docker compose up --build -d drait-frontend
```
