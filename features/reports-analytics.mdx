# Dashboard, Reports, and Analytics

DRAIT Mini-MES provides comprehensive analytics through the executive dashboard and productivity reports, enabling data-driven decision making and performance monitoring.

## Executive Dashboard

The dashboard provides real-time operational visibility with KPIs, charts, and alerts.

### Accessing the Dashboard

Navigate to **Dashboard** from the main menu. The dashboard loads data for the selected time period (3, 6, or 12 months).

<Info>
  Dashboard data updates when you reload the page. KPIs reflect current system state at load time.
</Info>

### Time Period Selection

Control the analysis window using the period selector:

<CardGroup cols={3}>
  <Card title="3 Months" icon="calendar">
    Last quarter view for short-term trends.
  </Card>
  
  <Card title="6 Months" icon="calendar">
    Half-year analysis for medium-term patterns.
  </Card>
  
  <Card title="12 Months" icon="calendar">
    Full year view for annual performance (default).
  </Card>
</CardGroup>

Click the period buttons in the dashboard header to switch views. Charts and metrics recalculate instantly.

## Charts and Visualizations

The dashboard includes four interactive charts using Recharts library:

### 1. Production Trend (Line Chart)

**Shows:** Monthly closed order volume and average hours per order

**Data:**
- **OTs cerradas**: Count of work orders closed each month
- **Horas prom./OT**: Average hours worked per order

**Use case:** Identify seasonal patterns, capacity trends, and efficiency changes over time.

<Tabs>
  <Tab title="Interpretation">
    - **Rising closed orders**: Increasing production volume
    - **Rising avg hours**: Decreasing efficiency or more complex jobs
    - **Declining avg hours**: Improving efficiency or simpler jobs
    - **Seasonal peaks**: Plan capacity for high-demand periods
  </Tab>
  
  <Tab title="Data Source">
    ```typescript
    // Backend aggregation
    const lineData = monthlyManagement.map((m) => ({
      mes: m.monthLabel,       // e.g., "Ene 2026"
      closedOrders: m.closedOrders,
      avgHours: m.avgHours
    }));
    ```
  </Tab>
</Tabs>

### 2. Order Status Distribution (Pie Chart)

**Shows:** Current work order breakdown by status

**Data:** Percentage distribution across:
- Pendiente
- Planificada
- En proceso
- Pausada
- Finalizada
- Entregada
- Cancelada
- Retrabajo

**Use case:** Understand current pipeline composition and identify bottlenecks.

<Info>
  A high percentage in "En proceso" or "Pausada" may indicate resource constraints or blockers.
</Info>

### 3. Revenue vs Material Cost (Bar Chart)

**Shows:** Monthly comparison of estimated revenue and actual material costs

**Data:**
- **Ingreso estimado**: Sum of `estimatedCost` for closed orders
- **Costo materiales**: Sum of material consumption costs

**Use case:** Track margin trends and identify months with high material costs.

### 4. Monthly Margin (Stacked Bar Chart)

**Shows:** Monthly profit/loss breakdown

**Data:**
- **Ganancia** (green): Positive cost deviations
- **Pérdida** (red): Negative cost deviations

**Use case:** Identify profitable vs unprofitable months, spot patterns in overruns.

## Understanding Deviations

### Time Deviation

Measures actual hours vs estimated hours:

```typescript
timeDeviationPct = ((workedHours - estimatedHours) / estimatedHours) * 100
```

**Examples:**
- Estimated 10h, worked 12h = **+20%** deviation
- Estimated 10h, worked 8h = **-20%** deviation (under time, positive outcome)

**Causes of high time deviation:**
- Inaccurate initial estimates
- Unexpected complexity
- Rework or quality issues
- Equipment downtime
- Material delays

### Cost Deviation

Measures actual material cost vs estimated cost:

```typescript
costDeviationPct = ((realMaterialCost - estimatedCost) / estimatedCost) * 100
```

**Examples:**
- Estimated $1000, spent $1200 = **+20%** deviation
- Estimated $1000, spent $900 = **-10%** deviation (cost savings)

**Causes of high cost deviation:**
- Material price increases
- Higher scrap/waste than planned
- Specification changes
- Rework material consumption
- Inaccurate quantity estimates

<Warning>
  Positive deviations indicate overruns. Negative deviations can indicate good performance OR under-recording consumption.
</Warning>

## Dashboard Data Flow

<Steps>
  <Step title="User selects period">
    Period selector (3M, 6M, 12M) updates query parameter.
  </Step>
  
  <Step title="API request">
    Frontend calls:
    ```typescript
    GET /api/reports/dashboard?months=12
    ```
  </Step>
  
  <Step title="Backend aggregation">
    Backend queries database:
    - Count work orders by status
    - Sum costs and hours
    - Calculate deviations
    - Aggregate by month
    - Identify alerts
  </Step>
  
  <Step title="Response structure">
    Returns:
    ```typescript
    {
      kpis: { openCount, inProcessCount, delayedCount, ... },
      managementAnnual: { closedOrders, avgHoursPerOrder, revenueEstimated, ... },
      monthlyManagement: [{ monthKey, closedOrders, revenue, ... }],
      statusBreakdown: [{ status, count }],
      deviationAlerts: [{ id, code, severity, reasons }],
      recentOrders: [...],
      delayedOrders: [...],
      operatorLoad: [{ userId, fullName, count }]
    }
    ```
  </Step>
  
  <Step title="Frontend rendering">
    React components render:
    - KPI cards with formatted numbers
    - Recharts visualizations
    - Alert lists with conditionals
    - Tables with sorting/filtering
  </Step>
</Steps>

## Best Practices

<Card title="Dashboard Usage" icon="lightbulb">
  **Daily:**
  - Check alert counts in header
  - Review "Atrasadas" count for overdue work
  - Monitor "En proceso" to ensure progress
  - Review operator workload for balance
  
  **Weekly:**
  - Analyze production trend chart for capacity issues
  - Review alert details and assign corrective actions
  - Check status distribution for pipeline health
  
  **Monthly:**
  - Switch to 3M view for recent trends
  - Compare monthly margins to identify seasonal patterns
  - Review cost vs revenue gap
  - Analyze productivity report for top deviation causes
  
  **Quarterly:**
  - Use 12M view for annual performance
  - Review operator hours distribution
  - Adjust deviation thresholds if needed
  - Export data for external reporting
</Card>

<Card title="Deviation Management" icon="chart-line">
  **Proactive:**
  - Set realistic warning thresholds (20-25% time, 15-20% cost)
  - Monitor alerts daily, don't wait for critical
  - Document deviation causes in work order notes
  - Review quotation accuracy quarterly
  
  **Reactive:**
  - Investigate critical alerts immediately
  - Root cause analysis for repeated deviations
  - Update estimation formulas based on historical data
  - Train estimators on common overrun patterns
  
  **Long-term:**
  - Track deviation trends to validate threshold settings
  - Build estimation databases from actual performance
  - Implement continuous improvement on high-deviation patterns
</Card>
