## Overview

Client endpoints manage customer information and relationships. Clients are associated with quotations and work orders.

---

## GET /api/clients

Retrieve all clients for the company with pagination support.

**Authentication**: Required (JWT)

**Roles**: All authenticated users

### Query Parameters

<ParamField query="page" type="number">
  Page number (default: 1)
</ParamField>

<ParamField query="limit" type="number">
  Items per page (default: 50)
</ParamField>

<ParamField query="search" type="string">
  Search by client name, contact, or email
</ParamField>

### Response

Returns a paginated list of client objects.

<ResponseField name="data" type="array">
  Array of client objects
  
  <Expandable title="Client object properties">
    <ResponseField name="id" type="string">
      Client's unique identifier
    </ResponseField>
    
    <ResponseField name="companyId" type="string">
      Associated company ID
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Client's name or business name
    </ResponseField>
    
    <ResponseField name="contact" type="string">
      Contact person name
    </ResponseField>
    
    <ResponseField name="phone" type="string">
      Phone number
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Email address
    </ResponseField>
    
    <ResponseField name="address" type="string">
      Physical address
    </ResponseField>
    
    <ResponseField name="notes" type="string">
      Additional notes
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean">
      Whether the client is active
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of last update
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="number">
  Total number of clients
</ResponseField>

<ResponseField name="page" type="number">
  Current page number
</ResponseField>

<ResponseField name="limit" type="number">
  Items per page
</ResponseField>

<CodeGroup>
```bash cURL
curl -X GET "https://api.example.com/api/clients?page=1&limit=20&search=acme" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  'https://api.example.com/api/clients?page=1&limit=20',
  {
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  }
);

const { data, total } = await response.json();
```
</CodeGroup>

## GET /api/clients/:id

Retrieve a specific client by ID.

**Authentication**: Required (JWT)

**Roles**: All authenticated users

### Path Parameters

<ParamField path="id" type="string" required>
  Client ID
</ParamField>

### Response

Returns a single client object with all fields.

<CodeGroup>
```bash cURL
curl -X GET https://api.example.com/api/clients/clx1234567890 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  `https://api.example.com/api/clients/${clientId}`,
  {
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  }

const client = await response.json();
```
</CodeGroup>

## POST /api/clients

Create a new client.

**Authentication**: Required (JWT)

**Roles**: `DUENO`, `SUPERVISOR`, `ADMIN`

### Request Body

<ParamField body="name" type="string" required>
  Client's name or business name (max 120 characters)
</ParamField>

<ParamField body="contact" type="string">
  Contact person name (max 120 characters)
</ParamField>

<ParamField body="phone" type="string">
  Phone number (max 50 characters)
</ParamField>

<ParamField body="email" type="string">
  Email address (max 120 characters)
</ParamField>

<ParamField body="address" type="string">
  Physical address (max 250 characters)
</ParamField>

<ParamField body="notes" type="string">
  Additional notes (max 500 characters)
</ParamField>

### Response

Returns the created client object.

<CodeGroup>
```bash cURL
curl -X POST https://api.example.com/api/clients \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "ACME Corporation",
    "contact": "John Doe",
    "phone": "+1-555-0123",
    "email": "john@acme.com",
    "address": "123 Main St, City, State 12345",
    "notes": "Preferred customer, net 30 payment terms"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.example.com/api/clients', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${accessToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'ACME Corporation',
    contact: 'John Doe',
    phone: '+1-555-0123',
    email: 'john@acme.com',
    address: '123 Main St, City, State 12345'
  })
});

const client = await response.json();
```
</CodeGroup>

## PATCH /api/clients/:id

Update an existing client.

**Authentication**: Required (JWT)

**Roles**: `DUENO`, `SUPERVISOR`, `ADMIN`

### Path Parameters

<ParamField path="id" type="string" required>
  Client ID to update
</ParamField>

### Request Body

All fields are optional. Only include fields you want to update.

<ParamField body="name" type="string">
  Client's name (max 120 characters)
</ParamField>

<ParamField body="contact" type="string">
  Contact person name (max 120 characters)
</ParamField>

<ParamField body="phone" type="string">
  Phone number (max 50 characters)
</ParamField>

<ParamField body="email" type="string">
  Email address (max 120 characters)
</ParamField>

<ParamField body="address" type="string">
  Physical address (max 250 characters)
</ParamField>

<ParamField body="notes" type="string">
  Additional notes (max 500 characters)
</ParamField>

### Response

Returns the updated client object.

<CodeGroup>
```bash cURL
curl -X PATCH https://api.example.com/api/clients/clx1234567890 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+1-555-0124",
    "notes": "Updated payment terms to net 45"
  }'
```
</CodeGroup>

## DELETE /api/clients/:id

Soft delete a client (sets `isActive` to false).

**Authentication**: Required (JWT)

**Roles**: `DUENO`, `SUPERVISOR`, `ADMIN`

### Path Parameters

<ParamField path="id" type="string" required>
  Client ID to delete
</ParamField>

### Response

Returns the deactivated client object.

<CodeGroup>
```bash cURL
curl -X DELETE https://api.example.com/api/clients/clx1234567890 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```javascript JavaScript
const response = await fetch(
  `https://api.example.com/api/clients/${clientId}`,
  {
    method: 'DELETE',
    headers: {
      'Authorization': `Bearer ${accessToken}`
    }
  }
);

const result = await response.json();
```
</CodeGroup>

<Note>
  This is a soft delete operation. The client record remains in the database with `isActive` set to `false`. This preserves historical data integrity for quotations and work orders.
</Note>
